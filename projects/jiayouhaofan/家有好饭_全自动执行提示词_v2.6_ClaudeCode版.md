1. **禁止回问。** 全程不得向委托方提出任何问题、不得请求确认、不得输出"请提供更多信息"类语句。

5. **止损条款(自动执行):** ①的结论为"不建议立项"→仅输出①的完整报告与否决理由及替代切口,终止②③④;"有条件立项"→列明条件清单后继续执行②③④,但每个阶段开头必须重申前提条件;"建议立项"→完整执行。注意:因"资质全部按合规"已是预设,**资质缺口本身不直接构成否决**——应转化为①中"合规配置清单"的成本、周期与经营边界;只有当合规配置的成本 / 周期 / 内容边界收窄严重到击穿商业模式或预算时,才据此给出"有条件立项"或"不建议立项"。
6. **独立性条款:** 客户付费购买的是准确判断,不是乐观结论。"不建议立项"是合格交付物,迎合性结论是不合格交付物。

## 判断稳定性框架(无人工参与下保证结论可信、可复现的核心机制)

自由发挥式的判断在不同执行之间会漂移。本框架把关键判断固化为**规则驱动**:同样的证据应导出同样的结论,结论错了也能沿着规则追溯到哪一环出了错。

### 1. 关键术语操作化定义(判断前先适用定义,禁止自由解释)

- **"真需求 / 强需求"成立** = 同时满足:(a) A 级数据显示目标人群规模与问题发生率达到千万级;(b) 深度账号研究中至少 2 个账号被观察到用户为同类服务/产品真实付费的 E1 或相互印证的 E2 证据;(c) 需求由用户主动搜索或主动咨询行为支撑(搜索指数、评论区求助密度等可观察迹象)。缺任一项,降级为"泛知识消费倾向,需一线验证"。
- **"起号窗口存在"** = 近 6 个月内可观察到该细分方向有新账号(非存量大号)起量的 E1/E2 案例 ≥2 个。找不到即判"窗口存疑",不得以"市场很大所以有机会"替代。
- **"价格带成立"** = 深度账号研究中观察到同客群、同量级价格的产品有真实销量代理指标;无对标观察的价格档一律标注"未经市场验证"。
- **"单位经济成立"** = 保守情景(见第 4 条)下,单客毛利 > 单客获取成本 + 单客交付成本,且营养师工时不超载。

### 2. 立项评分卡(①的三选一结论由评分卡分档产生,不由行文语气产生)

| 维度 | 权重 | 1 分锚定 | 3 分锚定 | 5 分锚定 |
|---|---|---|---|---|
| 合规可行性 | 25% | 合规配置后仍无法面向目标客群讲核心话题 | 可讲但边界明显收窄,需内容降级 | 配置完成后核心话题基本可讲 |
| 需求强度 | 15% | 仅泛知识消费迹象 | 满足操作化定义 2/3 项 | 满足全部 3 项 |
| 流量窗口 | 15% | 无新号起量案例,供给明显过剩 | 有 1—2 个案例但路径特殊 | ≥2 个可复制路径的新号案例 |
| 单位经济 | 20% | 保守情景为负且无改善路径 | 保守情景微利或平衡 | 保守情景明确为正 |
| 竞争与差异化 | 10% | 差异化点已被对标账号占据 | 有差异化但信任门槛高 | 有对标验证过的空位 |
| 团队/预算匹配 | 15% | 200 工时或 215 万明显不够跑完验证 | 紧张但可通过砍任务达成 | 从容覆盖 90 天验证 |

每个维度打分必须引用具体证据(注来源与证据等级)。加权总分:**≥4.0 → 建议立项;3.0—3.9 → 有条件立项;<3.0 → 不建议立项**。
**一票否决(覆盖总分):** (a) 合规可行性 ≤2 分 → 最高只能"有条件立项";(b) 单位经济保守情景为负且无可信改善路径 → "不建议立项";(c) 深拆账号 E1 占比 <40% 且关键 E2 来源相互矛盾 → 最高只能"有条件立项",条件为"补充一线验证"。

### 3. 结论卡片(①的 7 个核心判断与最终总结论,一律用此格式输出)

> 【结论】一句话,明确无模糊 |【评分/档位】(适用时) |【关键依据】≤3 条,每条注来源编号与证据等级 |【反方最强论点及回应】见第 5 条 |【敏感假设】指向假设登记表编号 |【置信度】高 / 中 / 低

置信度标尺:**高** = ≥2 个独立来源(不同机构且非同源转载)支撑且反方论证不成立;**中** = 证据部分支撑或来源独立性不足;**低** = 以 E3 推断为主。置信度为"低"的结论不得作为唯一依据支撑立项档位。

### 4. 单位经济三情景模型(②③的一切财务估算必须走此模型)

对 99 / 399 / 2999 三级产品分别建立 保守 / 基准 / 乐观 三情景:每个情景的转化率、完课率、复购率等参数取值必须注明来源(对标观察 / 行业估算 / 预设假设),公式完整展示,禁止直接给结果数。**立项结论只允许建立在保守情景之上**;基准与乐观情景仅用于展示上行空间。

### 5. 红队反方论证(强制,两处)

在①出结论前、以及最终总结论前,各执行一次角色切换:以"受雇论证此项目不该做(或该做,取与当前倾向相反的立场)"的独立顾问身份,写出最强反方论证(≥5 条,每条须有证据或逻辑支撑,不许稻草人)。然后逐条回应:被驳倒的说明为什么,驳不倒的必须体现为评分下调、置信度下调或立项条件——**反方论证不允许"仅供参考",必须对结论产生可见影响或被显式驳回**。红队记录完整保留在报告附录。

### 6. 关键结论最低证据门槛

任何直接影响立项档位的结论,须有 ≥2 个相互独立的来源支撑(不同机构、非转载同源);只有单一来源的,结论措辞降为"单一来源显示……待印证",且该维度评分不得高于 3 分。

# 提示词① 合规前置核查 + 赛道可行性判断(含止损条款)

以独立分析师身份对项目做冷静的可行性判断,而非论证其可行。"劝退不该做的项目"与"确认该做的项目"同等有价值。

**研究方法总纲:本阶段的一切判断,必须以对实际运营中的主播 / 创作者账号的深度研究为实证地基。** 平台规则告诉你什么不能做,宏观数据告诉你需求存不存在,但只有正在运营的真实账号能告诉你:什么内容真的起量、什么信任姿势真的成立、什么转化动线真的在跑通、钱真的从哪里流进来。禁止先有结论再找例证;必须先完成深度账号研究,再基于研究发现回答核心判断。

## 第 0 问:资质与合规前置核查(先于一切其他分析)

分证书类型逐一核查并给出结论:

1. 各证书类型当前能否通过抖音的职业认证 / 健康类创作者认证?能通过哪一档?不能通过有什么内容限制与限流后果?
2. 各证书类型在抖音能否开橱窗并挂"健康服务 / 咨询 / 课程"类目?哪些类目对非医师资质关闭?

### 第一层:广度扫描(15—20 个账号)

覆盖六类:营养师 IP / 医生健康科普 / 减脂控糖餐·家庭餐桌 / 中老年养生 / 训练营·私域转化 / YouTube 中文健康饮食。每个账号建立基础画像行(表格):账号名、平台、粉丝量级、内容形式、更新频率、可观察的变现动作(橱窗 / 直播带货 / 课程挂载 / 引流私域方式)、活跃状态(近 30 天是否更新)。

### 第二层:深度拆解(6—8 个标杆账号,每个单独成节)

从广度样本中选出 6—8 个"正在真实赚钱迹象最明显"的账号(优先:有持续更新 + 有可观察变现动作 + 与本项目客群或模式相关),逐个完成**业务逻辑逆向拆解**,每个账号按以下五层结构输出:

1. **流量层:** 栏目结构与选题规律(近 30—50 条视频的选题分布)、爆款内容的结构复盘(开头钩子类型、时长、叙事结构、标题句式)、发布节奏、可观察的流量来源构成迹象(推荐 / 搜索 / 直播导流)。
2. **信任层:** 人设定位、资质与身份的展示方式(尤其注意:非医师身份如何合规地建立专业信任)、案例与用户见证的呈现手法、出镜方式与场景选择。
3. **转化层:** 完整还原其转化动线——从哪类内容、用什么钩子(资料包 / 测评 / 直播预约 / 粉丝群)、引导到什么载体(橱窗 / 小程序 / 企微 / 直播间);直播运营拆解(频率、时段、直播间结构、留人与逼单手法、挂载商品类型与价格带);记录其规避平台导流限制的**合规做法**(仅记录合规姿势,不研究违规绕行技巧)。
4. **产品层:** 产品阶梯与定价(引流品 / 主力品 / 后端品)、交付形态、可观察的销量区间(橱窗显示销量、课程显示学习人数等代理指标)。

**证据分级与降级协议(深度账号研究的核心纪律):**

抖音、视频号均为强登录墙生态,合规抓取手段可能无法直接观察账号主页、视频列表与直播间。为此,所有拆解结论必须标注证据等级:

- **E1(一手可核验):** 直接观察到的、附可直达链接的内容(可公开访问的视频页、账号页、橱窗页)。结论锚定"账号名 + 具体视频标题或直播日期 + 链接"。
- **E2(二手可署名):** 来自可署名的第三方拆解、行业媒体报道、数据平台公开榜单(注明作者 / 机构、发布时间、链接),并标注"未经一手验证"。多个独立 E2 来源相互印证的,注明印证关系。
- **E3(推断):** 基于 E1/E2 的合理推断,完整展示推断链条。

硬性规则:(a) 观察不到且无 E2 来源的维度写"未观察到",禁止用行业通识填空冒充该账号的事实;(b) B 级数据(真实转化率、销售额)在任何证据等级下都不给具体数字,只给代理指标;(c) 每个深拆账号的小节末尾标注其 E1/E2/E3 结论占比;**(d) 若全部深拆账号合计 E1 占比低于 40%,必须在执行摘要中醒目声明"账号研究以二手来源为主",并将据此得出的核心判断置信度整体下调一档、在结论中显式体现;** (e) 禁止为提高 E1 占比而使用绕过登录墙等违规手段——诚实的 E2 优于违规的 E1。

# 提示词④ 风险清单与合规红线深度分析

以合规顾问与风控顾问身份,基于①②③输出可贴在工作台上用的风险与合规文档,重点是可操作的自查清单与预警信号。开头重申前提条件(如有)。

## 任务一:发布前合规自查清单

每条视频 / 每场直播发布前逐项打勾:医疗化表达红线词自查(引用②清单)、疗效承诺自查、用户案例授权与脱敏、营养师资质展示规范(按①结论:能展示什么、不能自称什么)、私域导流话术合规、中老年用户保护(不制造健康焦虑、不诱导冲动消费、大额消费建议子女知情)。

## 任务二:风险矩阵

逐项分析:平台合规封禁 / 医疗化表达投诉 / 无资质认证导致持续限流 / 交付能力不足 / 营养师 IP 单点依赖 / 内容同质化(含 AI 批量养生号冲击) / 起流量不赚钱 / 私域运营过重 / 中老年用户被误导的舆情与监管风险 / 预算超支。

每项:发生概率(高中低) / 影响程度 / 可观察可量化的预警信号 / 预防动作 / 发生后第一响应步骤。表格呈现。

## 任务三:止损线

3 条硬性止损线(合规处罚等级 / 90 天数据下限 / 预算消耗比例或营养师工时超载阈值),触发即调整或暂停,写明谁判断、如何执行。

# 最终整合输出要求

报告按以下结构一次性交付:

1. 假设登记表(自行设定的全部预设条件)
2. 执行摘要(800—1200 字,含最终立项判断)
3. ①合规核查与可行性判断(含深度账号研究:广度扫描表、6—8 个标杆账号逐个拆解、跨账号规律总结、数据来源)
4. ②定位与产品转化体系(仅当①未否决)
5. ③90 天计划、分工与指标(仅当①未否决)
6. ④风险与合规(仅当①未否决)
7. 总结论:是否建议立项(明确三选一,附评分卡;出结论前完成第二次红队反方论证)、第一阶段最关键 5 个动作、90 天后的继续 / 调整 / 停止标准
8. 附录:(a) 数据来源清单(编号 / 来源名称 / 链接 / 访问日期 / 支撑的结论)及"无法验证 / 待核实"事项汇总;(b) 方法论与检索日志(关键检索词、检索日期、命中与排除的来源及排除理由——用于结论复现与复核);(c) 两次红队反方论证完整记录;(d) 交付前自检记录

写作要求:专业、具体、可执行;所有结论说明依据;不确定信息明确标注;数据附来源链接并遵守"数据铁律";禁止空话与口号;语言面向创业决策而非宣传。报告写入 `家有好饭_赛道分析报告_v1.md` 单文件交付。

## 篇幅预算(防前重后轻)

报告正文目标 3 万—5 万字(不含来源清单),各部分占比:①含深度账号研究约 40%、②约 20%、③约 25%、④约 15%。任何一章偏离配比超过 10 个百分点,须在该章开头说明原因(如止损触发导致的正常收缩)。禁止①写爆而③④退化为条目式敷衍——③④的表格必须填实,不得以"参照上文"代替具体内容。

## 交付前自检(强制,自检结果作为报告附录)

合并最终报告前,逐项自检并在附录记录结果:

1. **数据铁律逐条自查:** 全文扫描,确认不存在无来源、无推算链条、未登记假设的数字;发现即修正或删除。
2. **链接抽验:** 从来源清单中随机抽取不少于 10 条链接实际访问,确认可达且内容支撑所引结论;失效链接标注"链接已失效,引用时点为 X"。
3. **证据等级复核:** 核对每个深拆账号的 E1/E2/E3 标注与占比声明是否一致;E1 占比低于 40% 时确认执行摘要已作声明、置信度已下调。
4. **判断—证据反向核对:** 对①的 7 个核心判断逐一反查:每个判断是否都引用了深度账号研究的具体发现?没有的补充或改写为"证据不足,倾向性判断"。
5. **假设登记表完整性:** 通读全文,凡出现"假设 / 预计 / 按 X 估算"字样处,核对是否已登记;漏登的补登。
6. **约束一致性:** 核对③的工时表合计不超过 200 小时/周、预算表合计不超过总预算、投流占比在承诺区间内。
7. **止损条款执行核对:** 确认报告结构与①的结论档位一致(否决则无②③④)。
8. **评分卡一致性复核:** 重算加权总分,核对档位与分档规则一致、一票否决条款已适用;每个维度打分的证据引用真实存在于来源清单。
9. **红队影响核对:** 逐条检查两次反方论证中"未被驳倒"的论点,确认每条都已体现为评分下调、置信度下调或立项条件之一;发现"仅供参考"式悬空论点即为不合格,须回改。

自检发现的问题及修正记录一并写入附录——修正记录本身是交付质量的证明,不是瑕疵。
